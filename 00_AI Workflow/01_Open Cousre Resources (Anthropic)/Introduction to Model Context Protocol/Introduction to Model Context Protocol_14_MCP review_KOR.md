#### 요약

이제 MCP 서버를 구축했으므로 세 가지 핵심 서버 프리미티브를 검토하고 각각을 사용할 때를 이해해봅시다. 핵심 통찰력은 각 프리미티브가 애플리케이션 스택의 다른 부분에 의해 제어된다는 것입니다.

## 도구: 모델 제어

도구는 Claude에 의해 완전히 제어됩니다. AI 모델은 이러한 함수를 언제 호출할지 결정하고, 결과는 작업 달성을 위해 Claude에 의해 직접 사용됩니다.

도구는 Claude가 자율적으로 사용할 수 있는 추가 기능을 제공하는 데 완벽합니다. Claude에 "JavaScript를 사용하여 3의 제곱근을 계산해"라고 요청하면 Claude가 계산을 실행하기 위해 JavaScript 실행 도구를 사용하기로 결정합니다.

## 리소스: 앱 제어

리소스는 애플리케이션 코드에 의해 제어됩니다. 앱이 리소스 데이터를 가져올 때를 결정하고 사용 방식을 결정합니다. 일반적으로 UI 요소나 대화에 컨텍스트를 추가하는 데 사용됩니다.

우리 프로젝트에서는 두 가지 방법으로 리소스를 사용했습니다:

- UI의 자동 완성 옵션을 채우기 위해 데이터 가져오기
- 추가 컨텍스트로 프롬프트를 보강하기 위해 콘텐츠 검색

Claude의 인터페이스의 "Google Drive에서 추가" 기능을 생각해봅시다. 애플리케이션 코드는 어떤 문서를 표시할지 결정하고 채팅 컨텍스트에 해당 콘텐츠를 주입합니다.

## 프롬프트: 사용자 제어

프롬프트는 사용자 작업으로 트리거됩니다. 사용자가 버튼 클릭, 메뉴 선택 또는 슬래시 명령과 같은 UI 상호작용을 통해 미리 정의된 워크플로우를 실행할 시점을 결정합니다.

프롬프트는 사용자가 요청 시 트리거할 수 있는 워크플로우 구현에 이상적입니다. Claude의 인터페이스에서 채팅 입력 아래의 워크플로우 버튼은 프롬프트의 예제입니다. 미리 정의되고 최적화된 워크플로우로 사용자가 한 번의 클릭으로 시작할 수 있습니다.

## 올바른 프리미티브 선택

빠른 결정 가이드:

- **Claude에 새로운 기능을 제공해야 하나요?** 도구 사용
- **UI나 컨텍스트를 위해 앱으로 데이터를 가져와야 하나요?** 리소스 사용
- **사용자가 트리거할 수 있는 미리 정의된 워크플로우를 만들려고 하나요?** 프롬프트 사용

Claude의 공식 인터페이스에서 세 가지 프리미티브가 모두 실제로 작동하는 것을 볼 수 있습니다. 워크플로우 버튼은 프롬프트를 시연하고, Google Drive 통합은 리소스 작동을 보여주며, Claude가 코드를 실행하거나 계산을 수행할 때 내부에서 도구를 사용하고 있습니다.

이것들은 구체적인 사용 사례에 맞는 올바른 프리미티브를 선택하도록 돕는 높은 수준의 지침입니다. 각각은 애플리케이션 스택의 다른 부분을 서빙합니다. 도구는 모델을 서빙하고, 리소스는 앱을 서빙하고, 프롬프트는 사용자를 서빙합니다.
