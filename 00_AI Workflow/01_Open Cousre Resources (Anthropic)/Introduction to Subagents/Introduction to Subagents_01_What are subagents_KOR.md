Subagent(서브에이전트)는 Claude Code가 작업을 위임할 수 있는 전문화된 어시스턴트입니다. 특정 작업에 집중하는 도우미라고 생각하면 됩니다. 각 subagent는 자신만의 대화 컨텍스트 윈도우에서 실행되어 작업을 수행한 뒤 요약을 메인 스레드로 반환합니다. 파일 읽기, 검색, 도구 호출과 같은 중간 단계들은 모두 격리되어 메인 대화를 어지럽히지 않습니다.

## Subagent가 중요한 이유

Claude Code와 대화할 때마다 메인 컨텍스트 윈도우에 내용이 추가됩니다. 모든 도구 호출, 파일 읽기, 검색 결과가 그곳에 저장됩니다. 이 공간은 한정되어 있으며, 한 번 가득 차면 Claude는 대화의 앞부분을 놓치기 시작합니다.

Subagent는 별도의 컨텍스트 윈도우를 띄워 이 문제를 해결합니다. Subagent는 두 가지를 받습니다:

- **커스텀 system prompt(시스템 프롬프트)** -- 설정 파일에서 가져오며, subagent의 역할과 동작을 정의합니다
- **작업 설명(task description)** -- 사용자의 요청을 바탕으로 부모 에이전트가 작성합니다

그러면 subagent는 독립적으로 작업합니다. 파일을 읽고, 검색을 실행하고, 코드를 편집하는 등 필요한 모든 작업을 수행합니다. 완료되면 요약만 메인 대화로 돌아오고, 전체 subagent 대화는 폐기됩니다.

이렇게 하면 메인 컨텍스트가 깨끗하게 유지됩니다. 답을 찾기까지 거친 모든 과정의 잡음 없이 답만 얻을 수 있습니다. 단, subagent가 어떻게 결론에 도달했는지에 대한 가시성이 떨어진다는 트레이드오프가 있습니다.

## 실제 예제

낯선 코드베이스를 탐색 중이고 어떤 서비스가 환불을 처리하는지 알고 싶다고 가정해봅시다. Subagent 없이 Claude는 15개 파일을 읽고, 여러 차례 검색을 실행하고, 여러 함수 호출을 따라가게 될 수 있습니다. 필요한 것은 단 하나의 사실뿐인데도 이 모든 과정이 컨텍스트 윈도우를 채웁니다.

Subagent를 사용하면 경험이 훨씬 깔끔해집니다. 질문을 하면 Explore subagent가 띄워져, 자신의 컨텍스트에서 모든 탐색 작업을 수행한 뒤 초점이 잘 잡힌 답변을 돌려줍니다.

메인 컨텍스트 윈도우에는 질문과 요약만 기록되며, 그 과정에서 읽은 15개 파일은 기록되지 않습니다.

## 내장 Subagent

Claude Code는 즉시 사용할 수 있는 여러 내장 subagent를 제공합니다:

- **General purpose subagent** -- 탐색과 실행 모두 필요한 다단계 작업에 적합
- **Explore** -- 코드베이스의 빠른 검색과 탐색에 적합
- **Plan** -- plan 모드에서 계획을 제시하기 전에 코드베이스를 조사하고 분석하는 데 사용

## 커스텀 Subagent

내장 옵션 외에도 커스텀 system prompt와 도구 접근 권한으로 자신만의 subagent를 만들 수 있습니다. 이를 통해 코드 리뷰어, 테스트 작성자, 문서 생성기 등 워크플로에 맞춘 전문화된 에이전트를 필요한 만큼 정의할 수 있습니다.

## 핵심 요점

Subagent는 세 가지 주요 이점을 제공합니다:

- 작업을 초점이 잘 잡힌 단위로 나누어 각 subagent가 특정 작업에 집중하게 합니다
- 모든 중간 작업을 격리하여 메인 컨텍스트 윈도우를 깨끗하게 유지합니다
- 필요한 정보만 간결한 요약으로 돌려줍니다

내장 subagent를 사용하든 직접 만들든, subagent는 더 길어지는 Claude Code 세션을 더 잘 활용하기 위한 실용적인 방법입니다. 메인 컨텍스트의 잡음이 적을수록 더 오래, 더 효과적으로 작업할 수 있습니다.
