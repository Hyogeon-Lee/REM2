Claude Code는 당신의 코드베이스를 이해하고, 파일을 편집하고, 명령을 실행하며, 기존 개발자 도구와 통합되어 작업을 더 빠르게 완료할 수 있도록 도와주는 agentic 코딩 도구입니다. 터미널, Visual Studio Code, Claude Desktop 앱, 웹, JetBrains IDE에서 사용할 수 있습니다.

## Claude Code가 Claude와 다른 점

이전에 Claude.ai를 사용해본 적이 있다면 Claude Code가 무엇이 다른지 궁금할 수 있습니다. Claude.ai와 달리 Claude Code는 당신의 파일, 터미널, 전체 코드베이스에 직접 접근할 수 있습니다. 코드를 앞뒤로 복사하여 붙여넣는 대신, Claude Code가 직접 들어가서 작업을 수행합니다.

주요 차이점은 Claude Code가 **AI Agent** (에이전트)로 작동한다는 것입니다.

## Agent란 무엇인가?

AI Agent는 환경과 상호작용하고 정의된 목표를 완료하기 위해 행동을 수행할 수 있는 소프트웨어입니다. 핵심적으로 이는 대규모 언어 모델(LLM)이 실시간 루프에서 동작하는 방식으로 작동합니다. AI Agent는 목표 달성을 위해 도구, 외부 서비스, 또는 다른 AI Agent에 접근할 수 있습니다.

## Claude Code가 실제로 할 수 있는 일

실제로 어떻게 작동하는지는 다음과 같습니다:

- **코드베이스 읽기 및 이해.** Claude Code에 기능을 설명하거나 코드 전체에서 버그를 추적해달라고 요청할 수 있습니다.
- **프로젝트 전체에서 파일 편집.** Claude Code는 함수를 리팩토링하고 이를 참조하는 모든 파일을 업데이트할 수 있습니다.
- **터미널 명령 실행.** 빌드 스크립트를 실행하고, 테스트를 실행하고, 패키지를 설치하고, 출력을 사용하여 다음에 할 일을 결정할 수 있습니다.
- **웹 검색.** 문서나 최신 API 참조가 필요하면 이를 찾아볼 수 있습니다.

## Claude Code를 효과적으로 사용하기

Claude Code를 효과적으로 사용하려면 다음 세 가지 개념을 명심하세요:

**Context window (컨텍스트 창).** 이것을 Claude의 작업 메모리로 생각하세요. 많은 것을 보관할 수 있지만 한 번에 모든 것을 보관할 수는 없습니다. 이것은 "agentic" 측면이 들어오는 곳입니다. Claude는 전체 코드베이스를 컨텍스트에 로드하지 않고도 코드베이스 내에서 답변을 찾기 위한 전략적인 방법을 찾습니다.

**권한을 요청합니다.** 기본적으로 Claude Code는 명령을 실행하거나 변경하기 전에 당신에게 권한을 요청합니다. 직접 개입하는(hands-on) 방식을 선호하든, 맡겨두는(hands-off) 방식을 선호하든, 항상 통제권은 당신에게 있습니다.

**실수를 할 수 있습니다.** 다른 도구와 마찬가지로 Claude Code도 완벽하지 않습니다. 당신의 의도를 잘못 이해하거나, 버그를 도입하거나, 솔루션을 과도하게 설계(over-engineer)할 수 있습니다. 작업 루프에 함께 머무르면 이런 문제를 조기에 발견할 수 있습니다.

## 요약

Claude Code는 agentic 코딩 도구입니다. 코드베이스를 읽고, 파일을 편집하고, 명령을 실행하고, 외부 도구와 연결되어 더 빠르게 배포할 수 있도록 도와줍니다. 지금 바로 터미널, VS Code, JetBrains, Claude Desktop 앱에서 사용할 수 있습니다.
