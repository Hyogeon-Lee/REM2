## Claude 데스크톱 앱 탐색하기: Chat, Cowork, Code

Claude 데스크톱 앱은 Claude와 함께 작업하는 세 가지 방법을 제공합니다: Chat, Cowork, Code. 빠른 질문에서 복잡한 연구, 소프트웨어 구축까지 다양한 작업을 처리합니다.

Chat은 Claude.ai에서 알 수 있는 것과 동일하며, 컴퓨터에서 기본적으로 실행되는 빠른 항목, 스크린샷, 받아쓰기, 커넥터가 추가됩니다. Cowork는 에이전트 도구이므로, 목표를 제시하고 도구와 리소스를 연결하고 작업을 수행하도록 합니다. Cowork에서 Claude는 더 광범위하고 더 많은 여지가 있습니다. 이 더 넓은 범위는 더 철저한 연구 및 분석을 수행할 수 있게 하고 더 복잡한 문서 및 결과물을 생성할 수 있게 합니다. Code는 코드 작성 및 테스트에서 배포까지 소프트웨어를 구축하기 위한 것입니다.

Cowork와 Code는 동일한 엔진에서 실행됩니다. 둘 다 Claude Code 아래에 있으므로, 머신에 로컬이고, 독립적인 작업이 가능하며, 하위 에이전트를 시작하고 장시간 작업을 유지할 수 있습니다. 이를 통해 Claude는 연구 및 작성이나 소프트웨어 구축과 같은 더 큰 작업을 자체적으로 해결할 수 있습니다.

각 모드는 제공하는 업무 주위에 설계되어 있으며, 중요한 것을 보여주고 필요한 곳에서 제어를 제공합니다.

Chat은 질문을 하고, 아이디어를 브레인스토밍하고, 작성하거나, 앞뒤로 문제를 해결할 때 뛰어납니다.

Claude.ai를 사용해본 경우, 이것은 동일한 방식으로 작동하며 컴퓨터에서 기본적으로 실행되는 몇 가지 추가사항이 있습니다:

- **빠른 항목.** Mac에서 Option 키를 두 번 탭하여 작업 중인 것 위에 Claude를 불러냅니다. 앱 사이를 전환하면서 맨 위에 머물러 있는 컴팩트한 윈도우에서 응답합니다. 작업하는 것을 떠날 필요가 없습니다.
- **스크린샷 및 윈도우 공유.** 스크린샷을 캡처하거나 윈도우를 공유하여 Claude가 정확하게 보고 있는 것을 확인하게 합니다. 화면의 것을 설명하는 것보다 더 빠르고 더 정확합니다. (Mac)
- **받아쓰기.** 타이핑하는 대신 문제를 말해주세요. 큰 소리로 생각하고 있을 때, 키보드에서 떨어져 있을 때, 또는 말하는 것이 작성하는 것보다 빠른 곳에서 작업할 때 유용합니다. (Mac)
- **데스크톱 커넥터.** 커넥터를 통해 로컬 도구 및 서비스를 연결하여 Claude가 머신의 다른 도구와 함께 작동할 수 있게 합니다.

**시도해야 할 때:**

- 낯선 대시보드를 마주하고 있습니다. Option을 두 번 탭하고, 커서를 윈도우 위로 끌어 스크린샷을 만들고, "이 메트릭은 무엇을 의미합니까?"라고 물어보세요. Claude는 대시보드가 보기에 유지되는 동안 오버레이에서 답변합니다.
- 회의 사이에 있으며 프레젠테이션을 구조화하는 방법을 생각해보고 싶습니다. 빠른 항목을 열고, 음성으로 전환하고, 말해주세요. Claude는 당신이 말한 것으로부터 개요를 작성합니다.
- Apple Notes에서 수주 동안 제품 출시에 대한 아이디어를 수집했습니다. Settings에서 Notes 커넥터를 추가하고 Claude에게 요청하세요: "Osprey 출시에 대해 Notes의 모든 것을 함께 가져오세요. 내가 중단한 곳을 파악하고, 내 다른 연결된 도구에서 공백을 채우는 모든 것을 확인하세요." Claude는 머신에서 메모를 읽고, 가진 것을 조합하고, 당신이 중단한 곳에서 따릅니다.

## Cowork

Claude Cowork는 진정한 노력이 필요한 업무를 위해 만들어졌습니다: 많은 소스에서 정보를 가져오고, 이해하고, 완성된 것을 생성합니다.

Cowork에서 Claude는 멀티태스킹할 수 있으므로, 프로젝트의 다양한 부분을 동시에 처리할 수 있으므로 더 많은 소스를 가져올 수 있는 범위와 작업을 완료할 수 있는 체력이 있습니다. 철저한 연구 브리프, 교차 소스 재무 분석, 종단 간 계약 검토, 다양한 소스에 산재한 자료에서 만든 광택난 슬라이드 데크입니다.

시작하기 전에 Claude는 필요한 것을 파악하기 위해 짧은 질문 세트를 종종 요청합니다: 범위, 형식, 제약. 사이드바에서 검토할 수 있는 계획을 구축합니다. 작업하면서 작업이 함께 진행되는 것을 확인합니다: 가져오는 소스, 형태가 잡히는 파일, 계획을 통한 진행 상황. 여러 작업을 한 번에 실행할 수 있으며, 각각은 자신의 대화에 있고, 사이드바에서 전환할 수 있습니다.

- **폴더 액세스.** 컴퓨터의 폴더를 Claude에 가리키면 거기의 것을 읽고, 관련된 것을 파악하고, 완성된 작업을 같은 장소에 저장합니다. 또한 파일을 업로드하거나, 콘텐츠를 대화에 붙여넣거나, Claude가 필요한 것을 가져오는 도구를 연결할 수 있습니다.
- **예정된 작업.** Claude는 정기적으로 작업을 처리할 수 있습니다: Slack과 캘린더에서 끌어오는 일일 브리핑, 배송된 것의 주간 정리, 관심이 필요한 것을 정렬하는 아침 받은 편지함 분류. 작업과 실행 시기를 정의하면 Claude가 자동으로 각 시간을 처리합니다. 컴퓨터나 앱이 닫혀 있었다면 당신이 돌아오면 따라잡습니다.
- **하위 에이전트.** Claude가 작업의 일부를 병렬로 처리하도록 시작하는 배경 작업자. 복잡한 것을 요청하면, 예를 들어 여러 소스에서 가져오는 연구 브리프, Claude는 이를 하위 작업으로 분해하고, 각각에 고유한 컨텍스트가 있는 하위 에이전트를 할당하고, 결과를 조정하여 완성된 결과물을 하나 제공합니다.
- **발송.** 휴대 전화에서 Cowork 대화를 계속할 수 있는 지속적인 대화 스레드. Claude 모바일 앱에서 컴퓨터의 모든 것 (파일, 커넥터, 플러그인, 데스크톱 앱까지)을 사용하는 작업을 Claude에게 건넬 수 있습니다. 이 기능을 사용하려면 데스크톱과 모바일 앱이 모두 필요하며, 컴퓨터가 깨어 있고 데스크톱 앱이 열려 있어야 합니다.
- **Project (프로젝트).** Cowork의 프로젝트를 사용하면 관련 작업을 고유한 파일, 컨텍스트, 지침, 메모리가 있는 전용 워크스페이스로 그룹화할 수 있습니다. Claude에서 프로젝트를 사용하면 Cowork 프로젝트가 유사하게 작동하지만 데스크톱에 로컬로 존재하고 Cowork를 통해 실행하는 작업 주위에 구축됩니다.
- **브라우저 사용.** Chrome에서 Claude를 연결하면 Claude는 웹 사이트를 탐색하고, 페이지와 상호 작용하고, 발견한 것을 직접 작업 중인 작업에 가져올 수 있습니다. Cowork는 이렇게 10개 사이트에서 경쟁사 가격을 확인하거나 API가 없는 페이지에서 데이터를 수집하는 작업을 수행합니다.
- **컴퓨터 사용.** Claude가 필요한 것의 커넥터나 플러그인이 없으면 컴퓨터를 직접 탐색할 수 있습니다. 클릭하고, 입력하고, 앱을 열 수 있습니다. 정확히 하듯이 말입니다. Claude는 우선순위 순서를 따릅니다: 커넥터 먼저, Chrome, 그 다음 화면 상호 작용이므로, 항상 가장 빠르고 신뢰할 수 있는 경로를 선택합니다. Claude가 각 앱에 액세스하기 전에 권한 프롬프트가 표시되며, 제한하고 싶은 항목에 대해 블록리스트를 설정할 수 있습니다. 컴퓨터 사용은 Pro 및 Max 요금제에서 연구 미리보기 중이며, macOS 전용입니다. (Windows 곧 출시 예정)
- **플러그인.** 플러그인은 Claude가 자체적으로 가지지 않은 기능을 제공합니다: 라이브 금융 데이터 끌어오기, 회사의 내부 정보 기반 검색, 또는 특정 규정 준수 프레임워크 내에서 작업. Cowork 인터페이스에서 이들을 탐색하고 추가하여 작업에 맞게 조정하세요.
- **보호된 환경.** Cowork는 컴퓨터의 포함된 공간에서 실행됩니다. Claude는 공유한 폴더 내에서 파일을 읽고, 생성하고, 편집할 수 있지만, 그들 외부의 무엇이든 액세스할 수 없습니다.

**시도해야 할 때:**

- 데이터베이스인 것처럼 모든 도구를 쿼리하기를 원합니다. "마지막 분기에 가격 책정에 대해 결정한 것을 회의 메모, Slack, 이메일에서 검토한 후 Q3 데크를 결과로 업데이트해주세요?"라고 물어보세요. Cowork는 회의 메모, 슬라이드 데크, 이메일, Slack 스레드에서 답변을 찾습니다.
- 새로운 시장을 연구하고 경쟁사를 범위 지정하고 도구를 평가하고 있습니다. 여러 탭에 걸쳐 있고 정보를 추출하기 어려운 연구의 경우 Cowork는 사이트를 방문하고, 보고서를 읽고, 가격을 끌어오고, 소스와 함께 구조화된 브리프를 제공합니다. 단일 브라우저 탭을 열 필요가 없습니다.
- 계약, 재무 보고서, 회의 기록을 포함한 50개 이상의 프로젝트 문서 폴더를 해결해야 합니다. Cowork에게 이니셔티브와 가장 관련된 문서를 찾고 요약 메모를 생성하도록 요청할 수 있습니다. Cowork는 모든 페이지를 읽고, 전체 세트 전체에서 교차 참조하고, 모두 읽는 것에서만 나타나는 패턴을 끌어냅니다. 50개를 5개처럼 검토하세요.
- 매일 아침 같은 일을 계속하고 있습니다. 메시지 확인, 상태 업데이트 끌어오기, 당일 회의 준비. 한 번 설정하여 예정된 작업으로 설정하면 Claude가 반복해서 처리하므로 답변 대신 행정으로 시작하는 날이 됩니다.

Cowork는 Pro, Max, Team, Enterprise 사용자에게 사용할 수 있으며 정기적으로 새 기능이 추가되고 있습니다.

## Code

Code 탭은 Claude Code의 힘에 액세스할 수 있으므로 데스크톱 앱 내에서 직접 실행됩니다. 이를 통해 소프트웨어를 구축하기 위한 완전한 개발 환경을 얻을 수 있습니다.

Code를 통해 Claude는 코드베이스에 직접 작동합니다: 거기에 있는 것을 읽고, 코드를 작성 및 수정하고, 명령을 실행합니다. 시각적 diff는 변경된 것을 보여주고, 내장된 터미널은 명령을 실행할 때 보여주고, git은 모든 버전을 추적하여 항상 롤백할 수 있습니다.

Cowork가 공유한 폴더로 제한되는 포함된 워크스페이스에서 실행되는 위치에서 Code는 전체 파일 시스템, 터미널, 개발 도구에 직접 액세스하여 프로젝트에서 실행됩니다.

작업이 일어나는 곳을 선택합니다:

- **로컬:** 컴퓨터의 폴더를 선택하면 Claude가 해당 파일과 직접 작동합니다. 머신에서 실행되므로 Claude는 프로젝트를 읽고, 로컬 도구에 액세스하고, 브라우저에서 미리 볼 수 있는 개발 서버를 실행할 수 있습니다.
- **원격:** GitHub 저장소를 연결하면 Claude가 클라우드 환경에서 작동합니다. 세션은 앱을 닫아도 계속되므로 큰 리팩터를 시작하고 나중에 다시 확인할 수 있습니다. 더 큰 코드베이스 또는 개발을 로컬 머신 밖으로 유지하려는 경우에 좋습니다.

세 가지 상호 작용 모드를 사용하면 Claude가 얼마나 많은 것을 자체적으로 수행할지 제어할 수 있습니다:

- **Ask (물어보기):** Claude는 모든 변경을 제안하고 승인을 기다립니다. 시각적 diff를 검토하고 수정되기 전에 받아들이거나 거부합니다.
- **Code (코드):** Claude는 파일 변경을 자동으로 적용하지만 터미널 명령을 실행하기 전에 확인합니다.
- **Plan (계획):** Claude는 무엇이든 만지기 전에 전체 접근 방식을 개략적으로 설명합니다. 전용 계획 뷰어를 사용하면 작업이 진행될 때 전략을 검토하고 다시 방문할 수 있습니다.

프로젝트 및 필터 상태 (활성 또는 보관됨) 및 환경 (로컬 또는 클라우드)에 걸쳐 여러 세션을 실행할 수 있습니다.

Code 탭은 Pro, Max, Team, Enterprise 사용자에게 사용할 수 있습니다.
