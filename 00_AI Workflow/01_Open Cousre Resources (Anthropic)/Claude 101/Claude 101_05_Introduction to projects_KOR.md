- 프로젝트의 지식 기반에 문서 및 파일 추가하기
- Claude의 행동을 안내하기 위한 효과적인 프로젝트 지침 작성하기

#### 주요 내용

- **Project는 자체 포함된 워크스페이스** (메모리, Chat 기록, 지식 기반, 사용자 정의 지침 포함)입니다. 이를 특정 업무 스트림을 위한 전용 환경으로 생각해보세요.
- **Project 지식은 Claude의 이해를 향상시킵니다.** Project 내의 모든 Chat에서 Claude가 참조하는 관련 문서를 업로드할 수 있게 합니다. 매번 같은 파일을 다시 업로드할 필요가 없습니다.
- **Project 지침은 Claude의 행동을 안내합니다.** 톤, 전문 수준, 응답 스타일 등을 지정할 수 있습니다. 이 지침은 프로젝트 내의 모든 대화에 적용됩니다.
- **Project는 자동으로 확장됩니다.** 지식 기반이 컨텍스트 제한에 접근할 때 Claude는 응답 품질을 유지하면서 용량을 최대 10배까지 확장하기 위해 Retrieval Augmented Generation (RAG) 모드를 원활하게 활성화합니다.
- **Claude for Work 사용자는 Project를 통해 협력합니다.** Project를 팀원과 공유하여 모두가 동일한 컨텍스트, 지침, 누적 지식으로부터 이점을 얻게 합니다.

### Project란 무엇입니까?

Project는 Claude가 참조해야 하는 지식을 저장하고, 특정 주제나 업무 영역 주위의 관련 Chat을 조직하고, 같은 공유 컨텍스트에 액세스해야 하는 팀원과 협력하기에 이상적입니다.

#### Project를 사용할 때

Project는 일회성 질문이 아니라 진행 중인 것을 작업할 때 특히 가치 있습니다. 다음을 포함하는 워크플로우가 있을 때 Project 생성을 고려하세요:

- **반복적으로 사용할 참고 자료** (회의 메모, 설문 결과, 보고서, 역사적 데이터 등)
- **Claude가 응답해야 하는 방법에 대한 일관된 요구 사항** (항상 형식 언어 사용, 항상 소스 인용, 항상 우리 템플릿 따르기)
- **여러 사람이 동일한 기반에서 작업해야 하는 팀 협력 요구 사항**

### 첫 번째 Project 생성하기

Project를 설정하는 것은 단 몇 분이 걸립니다. 시작 방법은 다음과 같습니다:

#### 1단계: Project 설정하기

1. 왼쪽 사이드바 위에 마우스를 올리고 "Project"를 클릭하거나 claude.ai/projects로 직접 이동합니다.
2. 오른쪽 상단에서 "+ New Project"를 클릭합니다.
3. Project의 설명적 이름을 지정합니다 (예: "Q4 마케팅 캠페인" 또는 "제품 문서").
4. 작업 중인 것에 대한 간단한 설명을 추가합니다. Claude는 이 설명을 직접 보지 않지만, 당신과 팀원이 Project의 목적을 이해하는 데 도움이 됩니다.
5. 가시성 설정을 선택합니다: 비공개로 유지하거나 조직과 공유합니다 (Claude for Work 사용자).

#### 2단계: Project 지침 추가하기

Project 지침은 이 Project의 모든 대화에서 Claude가 어떻게 행동해야 하는지 알려줍니다. "지침"을 클릭하여 지침 패널을 엽니다.

좋은 Project 지침은 일반적으로 다음을 포함합니다:

- **작업 중인 것에 대한 컨텍스트:** "이 Project는 우리의 B2B 소프트웨어 제품을 위한 마케팅 콘텐츠 생성을 위한 것입니다."
- **프로세스 지침:** "먼저 이 청중을 매료시킬 블로그 구조를 고려하고 초안을 작성합니다."
- **톤 및 스타일 환경설정:** "전문적이지만 대화체 톤을 사용하세요. 가능하면 전문 용어를 피하세요."
- **구체적인 요구 사항:** "항상 마케팅 복사본의 끝에 행동 호출을 포함하세요."

지침을 작성했으면 "지침 저장"을 클릭합니다. 이들은 이 Project의 모든 Chat에 적용되고 사용자 환경설정과 스타일과 함께 작동합니다.

또한 Project 지침을 사용하여 워크플로우를 자동화할 수 있습니다. 예를 들어 "회의 기록을 업로드할 때 이 템플릿을 사용하여 구조화된 요약을 만듭니다". 지침을 Claude의 이 Project에 대한 행동을 프로그래밍하는 것으로 생각해보세요.

#### 3단계: 지식 기반 구축하기

Project의 지식 기반은 Claude가 참조해야 하는 문서를 업로드하는 곳입니다. Project의 주 페이지의 오른쪽에서 파일 메뉴를 찾습니다.

콘텐츠를 추가하려면 "+" 버튼을 클릭합니다. PDF, DOCX, CSV, TXT, HTML 등을 포함한 다양한 파일 유형을 업로드할 수 있습니다. Google Drive에 연결하여 문서를 직접 연결할 수도 있습니다.

**업로드할 것:**

- 참고 문서 (브랜드 지침, 스타일 가이드, 템플릿)
- 배경 자료 (연구 보고서, 회의 메모, 요구 사항 문서)
- Claude가 모방하기를 원하는 업무의 예시
- 기술 문서 또는 사양

**Pro 팁:** 파일의 이름을 설명적으로 지으세요. Claude는 파일 이름을 사용하여 올바른 정보를 이해하고 검색하므로 "Q4-2024-Brand-Guidelines.pdf"는 "document1.pdf"보다 더 유용합니다.

#### Project가 대규모 지식 기반을 처리하는 방법

많은 콘텐츠를 업로드할 때 어떤 일이 일어나는지 궁금할 수 있습니다. Project는 Retrieval Augmented Generation (RAG)이라는 프로세스를 통해 많은 양의 콘텐츠를 자동으로 처리하도록 확장됩니다. 높은 수준에서 이는 Claude가 업로드된 문서의 가장 관련 있는 부분을 자동으로 찾고 사용할 수 있다는 의미이며, 어떤 파일을 보아야 하는지 알려줄 필요가 없습니다.

Project 지식이 컨텍스트 윈도우 제한에 접근할 때 Claude는 원활하게 RAG 모드를 활성화합니다. 모든 Project 콘텐츠를 한 번에 메모리에 로드하는 대신 Claude는 지능적으로 답변에 필요한 가장 관련 있는 정보를 검색합니다. 이는 응답 품질을 유지하면서 Project의 용량을 최대 10배까지 확장합니다.

Project가 RAG 활성화될 때를 나타내는 시각적 표시기가 표시되지만, 경험은 동일해야 합니다. 여전히 문서를 업로드하고, Claude와 Chat하고, 컨텍스트에 맞는 응답을 얻을 수 있습니다.

### Project 내에서 작업하기

Project를 설정한 후 Claude와 Chat을 시작할 수 있습니다. Project 내의 각 대화는 자동으로 지식 기반 및 Project 지침에 액세스할 수 있습니다.

### 협력 기능

Claude for Work (Team 및 Enterprise) 요금제의 사용자는 협력 기능을 통해 Project의 힘이 더욱 커집니다.

#### 권한 수준

Project를 공유할 때 세 가지 권한 수준 중에서 선택할 수 있습니다:

1. **Can view (볼 수 있음):** 구성원은 Project 콘텐츠를 보고, 지식에 액세스하고, Chat 할 수 있지만 변경할 수 없습니다. 논의 권한을 가진 읽기 전용 액세스로 생각해보세요.
2. **Can edit (편집할 수 있음):** 구성원은 완전한 협력 권한을 가집니다. 지침을 수정하고, 지식을 업데이트하고, 다른 구성원을 관리하고, 적극적으로 Project에 기여할 수 있습니다.
3. **Owner (소유자):** Project 생성자는 Project에 액세스할 수 있는 사람을 포함한 모든 것을 제어합니다. 특정 사람과 공유하거나 전체 조직에 Project를 표시할 수 있습니다.

#### Project 공유하기

Project를 공유하려면:

- 공유하고 싶은 Project를 엽니다
- Project 이름의 오른쪽에서 "공유 Project" 버튼을 클릭합니다
- 이름 또는 이메일을 사용하여 개별 구성원을 추가하거나 이메일 주소 목록을 복사하여 붙여넣습니다 (이 경우 Project는 "Shared with you" 섹션에 나타납니다)
- 또는 "[조직명]의 모든 사람"과 공유하여 Project를 Team 탭 내에서 검색 가능하게 합니다

팀 구성원은 Project를 공유할 때 이메일 알림을 받으며, "Shared with me" 탭에서 공유된 Project를 찾을 수 있습니다.

### 영감을 주는 Project의 예시

시작할 위치를 확실하지 않으세요? 다음은 여러 기능 전체의 일반적인 Project 유형입니다:

- **Q4 제품 출시:** 제품 사양, 경쟁 분석, 메시징 브레인스토밍 메모를 업로드합니다. Claude는 문의나 문서 초안에 이 컨텍스트를 염두에 두고 있습니다.
- **연구 지원:** 경쟁 검토, 사용자 연구 데이터, 고객 피드백을 중앙 집중합니다. Claude는 소스를 종합하고, 보고서를 작성하고, 권장 사항 전체에서 일관성을 유지하는 데 도움이 될 수 있습니다.
- **클라이언트 계정 허브:** 클라이언트의 브랜드 지침, 과거 결과물, 커뮤니케이션 기록을 한 곳에 유지합니다. 지침을 설정하여 Claude가 그들의 톤과 일치하고 제안이나 보고서를 작성할 때 특정 컨텍스트를 참조합니다.
- **이벤트 계획 워크스페이스:** 장소 계약, 스피커 약력, 참석자 데이터를 업로드합니다. Claude는 run-of-show 문서, 참석자 커뮤니케이션, 이벤트의 테마와 일관성을 유지하는 사후 이벤트 보고서를 생성하는 데 도움이 될 수 있습니다.
- **직무 설명 생성기:** 과거 직무 설명, 팀 차터, 내부 인원 요청 문서를 수집합니다. Claude와 함께 팀의 실제 업무와 문화를 반영하는 직무 설명을 작성합니다.

### Project의 모범 사례

Project에서 최대한의 이점을 얻으려면:

- **초점을 맞추고 시작한 후 확장합니다.** 모든 것을 위한 하나의 Project를 만들려고 하지 말고, 특정 사용 사례부터 시작합니다. 진행하면서 언제든 더 많은 콘텐츠를 추가할 수 있습니다.
- **지식 기반을 최신으로 유지하세요.** 오래된 문서는 오래된 응답으로 이어질 수 있습니다. Project 지식을 주기적으로 검토하고 업데이트합니다.
- **명확한 지침을 작성하세요.** 원하는 것에 대해 구체적으로 명시합니다. 모호한 지침은 일관성 없는 결과로 이어집니다.
- **문서의 이름을 설명적으로 지으세요.** (예: 'Q4-2025-Sales-Report.pdf'가 'report.pdf' 아님)하고 관련 파일을 함께 그룹화합니다. Claude는 파일 이름과 근접성을 사용하여 문서 간의 관계를 이해합니다.
- **문서의 이름으로 참조하세요.** 질문을 할 때 특정 문서를 언급하여 Claude가 검색 범위를 좁히는 데 도움을 줄 수 있습니다: "우리의 Q3 보고서에 기반하여 최고의 고객 우려 사항은 무엇이었습니까?"

### 단원 성찰

계속 진행하기 전에 고려해보세요:

- 영구적 컨텍스트를 가진 전용 Project로부터 이점을 얻을 수 있는 진행 중인 업무는 무엇입니까?
- Claude에게 정기적으로 다시 업로드하거나 다시 설명할 것으로 예상되는 문서는 무엇입니까?
- 팀을 사용하고 있다면, 공유된 지식과 지침으로부터 이점을 얻을 Project가 있습니까?
